Let's recap

/* shorthand */
animation: <custom-ident> <duration> <timing-function> <delay> <direction> <iteration-count> <fill-mode> <play-state>;
/* separated properties */
animation-name: <custom-ident>;
animation-duration: <time>;
animation-timing-function: <easing-function>;
animation-delay: <time>;
animation-direction: <direction>;
animation-iteration-count: <number>;
animation-fill-mode: <fill-mode>;
animation-play-state: <play-state>;
@keyframes <custom-ident> {
from { /* statements */ }
to { /* statements */ }
}
  • The @keyframes rule is used to create complex animations.
  • The animation property is more powerful and flexible than the transition property.
  • We can chain multiple animations together, and the order in which they are declared is important.
